Git spurningar og svör
1.  a) cd change directory og það er hægt að slá inn "cd " með bili og slá síðan inn möppu heitið.
	b) cd ~ fer einum til baka
	c) ls birtir allar möppur og verkefni sem að hægt er að fara inn á með cd.
	d) pwd sýnir í hvaða möppu eða hvar þú ert stadur og ert að vinna í.
	e) mkdir býr til nýja möppu á þeim stað sem að þú ert í möppuni.

2.  a) git clone clonar skránna sem að þú ert með skráða. afrita það sem að þú velur eftir á þú skrifar git clone
	b) git log birtir eða sýnir allar stór breytingar eða pull og push sem að hafa átt sér stað.
	c) git status fer yfir skrárnar hjá báðum eða öllum aðilum og birtir það sem er öðruvísi eða github kannast ekki við.
	d) git diff sýnir þér hvaða breytingar hafa átt sér stað á milli verkefna í meiri smáatriðum en git log.
	e) git checkout skiptir um stöðu sem að notandinn er staddur og fer með hann þar sem að hlutirnir byrjuðu.

3.  a) git diff
	b) git diff --staged
	c) git diff commit1 commit2

4. man allat fyrri útgáfur og gerir þér kleyft að fara alltaf til baka

5. fleirri enn einn geta unnið saman og án þess að fucka upp fyrir hvor öðrum / version control

6. Ég held að ég mun gera það í hvert skipti sem að ég mun taka við verkefni til að fá alltaf að sjá hvað er öðruvísi við sitt hvort verkefnið og fundið best leiðina til að blanda þeim saman.

7. WD er hvar þú ert að vinna í, SA leyfir þér að breyta endalaust eitthverju sem er til, R geymsla yfir eitthvað sem búið er að setja inn

8. Það er sniðugt að nota eitt branch fyrir hvert og eitt verkefni, svo að það sé hægt að sjá hvert og eitt verkefni inn á GitHub.
